### Round 1: Growth Loop Discussion

**Elena Verna:** "Let me break this down. For an API testing tool, the natural growth loops are:

1. **Shared collections** — Users create API test collections and share them with teammates. Teammate sees the tool, signs up. This is how Postman grew.

2. **Public documentation** — Users publish API docs generated by the tool. External developers discover your tool through the docs.

3. **'Tested with [Mindcraft]' badge** — Users embed a status badge in their repos showing their API test results. Every repo visitor sees it.

Option 2 is the strongest — it turns your users' work product into your acquisition channel."

**Paul Graham:** "Elena's right about the loop, but I'd push back on optimizing for it right now. At 800 users, you should be talking to users, not building growth features. Call 20 of your most active users this week. Ask them: 'How did you find us? Have you told anyone about us? Why or why not?' The answers will tell you which loop is already happening organically. Amplify that one."

**Pieter Levels:** "Both of you are overcomplicating this. The growth loop is simple: build in public. Tweet your user count, your features, your revenue. I grew Nomad List from 0 to 50K users primarily through Twitter. Your 800 users will share your tweets if you make them feel like insiders.

Also — shared collections is just a feature. Ship it in a weekend. Don't plan it for a quarter."

### Round 2: Parallel Execution Plan

**Elena:** "Good instinct. Here's the parallel play: PG does the qualitative (talk to users), I'll design the growth loop (shared collections), and Pieter ships the quick win (build in public).

For shared collections, the loop mechanics are:
- User creates a collection → generates a shareable link
- Link recipient sees the collection + 'Create your own' CTA
- New user signs up → creates their own → shares → loop continues

The key metric: what % of shared links result in a new signup? Start measuring this immediately."

**Paul Graham:** "Agreed on parallel execution. For the user interviews, here's exactly what to ask:

1. 'What were you using before us?'
2. 'What made you switch?'
3. 'Have you recommended us to anyone?'
4. 'What's the #1 thing you wish we did better?'

Do 20 calls. You'll see patterns by call 10."

**Pieter Levels:** "While they're doing strategy, here's your shipping list for this week:
1. Monday: Add a $19/month pro plan (shared collections = pro feature)
2. Tuesday: Tweet '800 developers use [product] to test their APIs. Here's why:'
3. Wednesday: Ship shared collections (it's a permalink + auth, takes a day)
4. Thursday: Post on Hacker News Show HN
5. Friday: Email your 800 users about the new features"
